# Environment And Secrets

Last verified against current code and provider docs on 2026-06-17.

Use this with `.env.example`. Do not commit `.env.local` or real provider secrets.
Only variables prefixed `NEXT_PUBLIC_` are browser-visible.

## Demo Seed

`npm run demo:seed` is re-runnable against local Supabase. It creates or updates:

- 5 local test users
- 1 active `miso` organization
- 15 published events
- 31 ticket categories
- available ticket inventory for those categories
- 4 sold buyer tickets and matching mini-site purchases
- 15 controller-event links
- 3 payout-ready Stripe seller fixture rows

A fresh reset followed by one seed leaves the buyer holding 4 tickets. A later
seed run tops available inventory back up without claiming more buyer tickets.

Local test credentials from the seed:

| Role       | Email                   | Password         | Use                                     |
| ---------- | ----------------------- | ---------------- | --------------------------------------- |
| Buyer      | `buyer@miso.local`      | `misobuyer`      | Storefront checkout, wallet, redemption |

| Admin      | `admin@miso.local`      | `misoadmin`      | Admin/organizer workspace coverage      |

## Required For Local App

| Variable                        | Secret? | Purpose                                                                                                                                           | How to get it                                                                                              | Cost                                                       |
| ------------------------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| `NEXT_PUBLIC_APP_URL`           | No      | Browser-visible app origin. Local default is `http://localhost:3002`.                                                                             | Set manually.                                                                                              | Free.                                                      |
| `APP_URL`                       | No      | Server-side canonical origin for absolute URLs. Used before `NEXT_PUBLIC_APP_URL` for non-local requests.                                         | Set manually to local or deployed app URL.                                                                 | Free.                                                      |
| `NEXT_PUBLIC_SUPABASE_URL`      | No      | Supabase API URL used by browser and server clients.                                                                                              | Local: `supabase status`. Hosted: Supabase project API settings.                                           | Local free. Hosted has Free and paid plans.                |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | No      | Browser-safe Supabase public key. New Supabase docs call this a publishable key; this repo still uses the legacy env name. RLS must protect data. | Local: `supabase status` publishable/anon key. Hosted: Supabase project API settings.                      | Local free. Hosted has Free and paid plans.                |
| `SUPABASE_SERVICE_ROLE_KEY`     | Yes     | Server-only Supabase admin key. Bypasses RLS and is used by seed scripts, webhooks, cron, and server mutations.                                   | Local: `supabase status` secret/service-role key. Hosted: Supabase project API settings or new secret key. | Local free. Hosted has Free and paid plans.                |
| `SUPABASE_DB_URL`               | Yes     | Direct Postgres connection string for CLI/type workflows and maintenance.                                                                         | Local default is in `.env.example`. Hosted: Supabase database connection string or pooler.                 | Local free. Hosted database usage may require a paid plan. |

Supabase note: the 2026 changelog says new projects can require explicit Data
API grants for new tables. If a new table works in SQL but not through
`supabase-js`, check Data API exposure grants in addition to RLS.
