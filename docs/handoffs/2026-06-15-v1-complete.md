## Get running (fresh clone)

```bash
npm install --legacy-peer-deps
cp .env.example .env.local      # fill values; minimal local = Supabase + Stripe
supabase start && supabase db reset
npm run supabase:types          # regenerates types AND re-appends the alias block
npm run dev                     # http://localhost:3002
```

The app is **env-gated**: AI, email, Redis, Sentry, and Vercel-domains all no-op cleanly without their keys, so local/CI stay green. AI needs `ANTHROPIC_API_KEY` (chat) + `OPENAI_API_KEY` (embeddings); see `.env.example` and `README.md`.

## Architecture map

- `src/app/` — App Router. Buyer surfaces under `/s/[organizationSlug]` (storefront), org-admin under `/admin` + `/smartboard` (settings), gate under `/redeem`. API routes under `src/app/api/`.
- `src/lib/stripe-marketplace/` — payments core (separate charges and transfers). Treat changes here as high-risk and verify deliberately. See ADRs 0001–0003.
- `src/lib/ai/` — env-gated AI client/embeddings/retrieval/indexing/prompts (ADR 0004). Retrieval is cross-org-isolated via `match_org_embeddings`.
- `src/lib/organizations/` — org context, auth (`getOrganizationRole`, `is_organization_admin`), public resolution, host routing.
- `src/lib/{email,rate-limit,supabase}/` — env-gated integrations + service/server clients.
- `src/components/landing/` + `src/components/motion/` — the marketing landing ("Operator Editorial").
- `supabase/migrations/` — timestamped, forward-only. `src/types/db.ts` is generated; convenience aliases live in `scripts/db-aliases.ts` (ALIAS_BLOCK) and are re-appended by `npm run supabase:types`.

## Conventions & gotchas

- **Branching:** PR every change against `development` (never `main`; never push `main`). `main` moves only when prod is cut from `development`.
- **Per change:** failing test first on non-trivial work; `typecheck`+`lint`+`test:unit`+`build` green before PR; ADR for hard-to-reverse choices (`prompts/adr.md`); update `docs/CONTEXT.md` for new domain terms; append `MEMORY.md`/`LESSONS.md` for non-obvious facts/heuristics.
- **Hooks** (`.codex/hooks/`): `protect-secrets.sh` blocks file tools on real secrets (`.env`, `.env.local`, `*.pem`, …) but **allows `*.example`**; `block-dangerous.sh` blocks `rm`/`mv`/truncate of `.env`/`.env.local` and `git clean -f` via shell hooks (append `>>`, `cp` restore, and `.env.example` are fine). Note: these can false-positive on commit/PR text that literally contains the patterns — reword the message, don't disable the hook.
- **Env-gating pattern:** optional integrations use a lazy singleton returning `null` when the key is absent; every public helper guards on it. Mirror this for any new external dependency.
- **Cross-org isolation is non-negotiable** in any multi-tenant read (esp. AI retrieval): resolve the org server-side; never trust a client-supplied org id for scope.
- **Lessons** worth knowing live in `LESSONS.md` (e.g. always `set -o pipefail` in piped verifications; macOS has no `timeout` — use the tool's own timeout; pre-stage fragile infra inline before fanning out workflow agents).
