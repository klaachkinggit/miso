# MisoTicket Receiver-Safe Transfers

MisoTicket is deployed per Event and backed by generated bytecode checked into the app. Changing it affects every newly deployed Event contract, so this records the small safety change made before Base mainnet.

## Decision

- Reject `address(0)` as the constructor admin.
- Store token URI before `_safeMint` so an ERC-721 receiver callback cannot observe an empty URI.
- Use `_safeTransfer` for backend-admin transfers so transfers to contracts without `onERC721Received` revert instead of stranding tickets.

## Why This Is Hard to Reverse

Existing deployed Event contracts keep their old bytecode. Only newly deployed Events receive this behavior, so the safest time to tighten the invariant is before more contracts are created.

## Consequences

- Personal-wallet export and resale transfers to non-receiver contracts now fail fast.
- Contract compile output must be regenerated after Solidity edits.
- Local coverage is source/bytecode regression coverage; the existing live Base Sepolia smoke remains the full EVM check when credentials are available.
