# Goal
Design from the smallest practical screen upward so essential workflows stay fast, focused, and touch-friendly.

# Instructions
1. Identify the mobile user's top task and remove secondary distractions from the initial view.
2. Design the narrow layout, touch targets, keyboard behavior, and scrolling model first.
3. Expand progressively for tablet and desktop with additional context, not duplicated content.
4. Handle mobile navigation, sticky actions, forms, modals, and media carefully.
5. Test thumb reach, tap spacing, viewport height changes, and virtual keyboard overlap.
6. Verify performance on slower mobile networks and devices when possible.

# Input
Use the target user, mobile workflow, content priority, breakpoints, and existing app navigation.

# Output
Generate mobile-first layouts, components, CSS, and responsive enhancements.

# Best Practices
- Keep critical actions reachable and labels readable.
- Avoid hover-only affordances.
- Use native inputs and mobile-friendly field types.
- Prevent horizontal scrolling unless it is a deliberate contained region.
- Keep page weight and blocking scripts low.
