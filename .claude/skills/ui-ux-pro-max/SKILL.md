# UI/UX Pro Max

Use when task changes how UI looks, feels, moves, or gets used.

## Apply

- Pages, dashboards, admin panels, SaaS, e-commerce, portfolio, mobile apps.
- Components: buttons, modals, nav, sidebars, cards, tables, forms, charts.
- Design calls: color, typography, spacing, animation, density, hierarchy.

Skip backend-only, infra, DB-only, non-visual scripts.

## Priority

1. Accessibility: contrast, labels, alt text, keyboard path, focus, reduced motion.
2. Touch/interaction: 44px+ targets, loading/error states, no hover-only critical actions.
3. Performance: optimized images, reserved dimensions, lazy load, no layout shift.
4. Product fit: one aesthetic, matched to domain.
5. Layout: mobile-first, no horizontal scroll, stable controls, safe spacing.
6. Type/color: readable scale, semantic tokens, accessible pairs, no tiny body text.
7. Motion: 150-300ms, purposeful, transform/opacity, interruptible.
8. Forms: visible labels, local errors, helper text, clear submit feedback.
9. Navigation: predictable back/deep links, not overloaded.
10. Data: legends/tooltips, tabular numbers, no color-only meaning.

## Miso Defaults

- Ops/SaaS surfaces: dense, restrained, scannable, repeat-action friendly.
- Public pages: strong first-viewport product signal, real assets, no fake filler.

Avoid blobs/orbs, generic gradients, nested cards, emoji icons, mixed icon styles,
viewport font scaling, negative letter spacing, and in-app text explaining UI features.
