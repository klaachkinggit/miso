# Awesome DESIGN.md

Use when UI task needs design-system direction, component patterns, or visual
language examples.

## Use

- Scan examples for patterns, not copy-paste.
- Extract useful tokens: spacing, type scale, layout density, motion, component states.
- Match product context. Miso ops/admin surfaces need dense, calm, scannable UI.
- Public brand pages can be more expressive, but still need real product signals.

## Avoid

- Generic gradients, blobs, fake dashboards, nested cards.
- Mixing multiple aesthetics.
- Importing external design rules wholesale without adapting to this app.
