# Goal
Improve conversion by reducing uncertainty, effort, and friction while keeping the experience honest and accessible.

# Instructions
1. Identify the conversion event, user intent, objections, and required trust signals.
2. Map the current or planned funnel and remove unnecessary steps, fields, and choices.
3. Make primary actions clear, specific, and visually dominant without hiding important terms.
4. Place proof, reassurance, pricing clarity, and support content near moments of hesitation.
5. Design validation, error recovery, loading, and abandonment states.
6. Measure the impact when analytics or tests are available.

# Input
Use the target flow, business goal, audience, analytics context, content, and constraints.

# Output
Generate conversion-focused pages, components, copy structure, forms, CTAs, and state handling.

# Best Practices
- Do not use dark patterns or misleading urgency.
- Minimize form fields and explain why sensitive data is needed.
- Keep checkout and signup steps visible and recoverable.
- Use clear error messages and preserve user input.
- Optimize performance on entry pages and transactional steps.
