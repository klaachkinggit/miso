# Goal
Design dashboard interfaces that help users scan, compare, decide, and act efficiently.

# Instructions
1. Identify core jobs, user roles, frequency of use, and the information hierarchy.
2. Prioritize dense but readable layouts over decorative marketing composition.
3. Use tables, filters, tabs, sidebars, charts, and detail panels based on workflow needs.
4. Define states for loading, empty, error, permission-limited, stale, and partially synced data.
5. Keep navigation predictable and actions close to the objects they affect.
6. Test the layout with realistic data volume, long names, and narrow desktop widths.

# Input
Use the domain model, metrics, user roles, tasks, data shapes, and existing app shell.

# Output
Generate dashboard pages, components, interaction states, data visualizations, and responsive layouts.

# Best Practices
- Avoid oversized hero sections inside operational products.
- Prefer clear density, alignment, and hierarchy over decorative cards.
- Use charts only when they help answer a question.
- Make destructive and high-impact actions explicit.
- Preserve keyboard, screen reader, and low-vision usability.
