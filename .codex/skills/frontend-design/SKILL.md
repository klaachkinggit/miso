# Frontend Design

Goal: non-generic, usable, polished UI with clear aesthetic direction.

## Process

1. Identify audience, primary task, domain constraints.
2. Pick one visual direction before coding.
3. Build actual usable screen, not marketing shell, unless user asks landing page.
4. Use real assets where useful.
5. Verify responsive layout and interaction states.

## Rules

- Match existing app conventions unless task asks redesign.
- Use icons for tools/actions; prefer lucide if available.
- No nested cards, decorative blobs, one-hue palettes, viewport-scaled text, negative letter spacing.
- Text must fit on mobile/desktop.
- Controls need stable dimensions and clear states.
- Operational SaaS UI: dense, quiet, scannable, repeat-action friendly.
