# Goal
Create distinctive, production-grade frontend interfaces that avoid generic AI aesthetics. Implement real working code with exceptional attention to aesthetic details, creative choices, and interaction quality.

# Instructions
1. Understand the interface purpose, audience, technical constraints, and primary user task before coding.
2. Commit to a bold aesthetic direction: brutally minimal, maximalist, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, soft/pastel, industrial, or another clear point of view.
3. Define what makes the work memorable: a layout idea, motion system, material treatment, visual metaphor, or unusually refined interaction.
4. Implement working code in the repo's stack and match existing component patterns unless the task explicitly calls for a new visual language.
5. Choose typography intentionally. Avoid default generic choices like Arial, Roboto, Inter, and system stacks unless the existing design system requires them.
6. Use CSS variables or design tokens for color, spacing, radius, shadows, and motion.
7. Build responsive states, empty/loading/error states, keyboard behavior, and accessible labels as part of the design, not afterthoughts.
8. Add motion where it has impact: page load choreography, scroll-triggered reveals, hover transitions, or spatial feedback. Keep it performant and respect reduced-motion preferences.
9. Refine spacing, alignment, contrast, hit targets, and text wrapping across mobile and desktop.
10. Verify the result in the browser when possible, including screenshots or visual checks for layout overlap and broken assets.

# Input
Use the user's frontend requirements, target audience, brand/product context, technical stack, existing app conventions, and any visual references or constraints.

# Output
Generate production-ready frontend code: pages, components, styles, assets, and interaction logic. Include only the files needed for the requested interface.

# Best Practices
- Avoid cookie-cutter patterns, purple-on-white gradients, generic SaaS cards, and decorative effects that do not serve the concept.
- Let implementation complexity match the aesthetic: maximalist designs need richer composition and motion; refined minimal designs need precision and restraint.
- Use semantic markup, accessible controls, sufficient contrast, visible focus states, and reduced-motion fallbacks.
- Keep visual systems cohesive: typography, color, layout, iconography, and motion should feel intentionally related.
- Prefer real assets, generated bitmap assets, or contextual visuals over abstract placeholder art when the subject needs to be understood.
