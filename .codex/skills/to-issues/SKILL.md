# To Issues

Source: https://github.com/mattpocock/skills

Convert a plan into issues that can be implemented independently.

Rules:

- Use vertical slices only; each issue delivers end-to-end value.
- Split anything larger than one focused session.
- Make dependencies explicit.
- Include context, acceptance criteria, and out-of-scope notes.
- Confirm before creating issues through a CLI or API.
