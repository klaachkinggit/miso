# Superpowers

Use this skill when the work is non-trivial, spans multiple files, or has unclear implementation order.

## Workflow

1. Brainstorm the implementation paths and identify the simplest viable one.
2. State the plan briefly before editing.
3. Execute in small steps.
4. Verify each behavior change with the narrowest useful test.
5. Stop expanding scope when the requested behavior is complete.

## Rules

- Ask one focused question when scope is ambiguous.
- Prefer existing project patterns over new abstractions.
- Keep changes reversible and reviewable.
- Do not stack this with another workflow skill.
