# UI/UX Pro Max

Use this skill when the work involves interface structure, visual direction,
interaction quality, accessibility, or responsive behavior.

## Workflow

1. Identify product type, audience, primary workflow, stack, and density needs.
2. Choose one coherent visual direction. Do not stack unrelated styles.
3. Define layout, navigation, hierarchy, spacing, typography, color, motion, and states before coding.
4. Build complete states: default, hover, focus, active, loading, empty, error, disabled, and mobile.
5. Verify accessibility: labels, keyboard flow, contrast, focus visibility, reduced motion, and semantic structure.
6. Review the rendered UI for text overflow, overlap, generic composition, weak hierarchy, and broken responsive behavior.

## Product Fit

- Operational tools: dense, calm, scannable, predictable, restrained.
- SaaS dashboards: clear navigation, fast comparison, compact controls, strong empty/error states.
- Commerce: product-forward, trust-building, direct purchase flow.
- Editorial/portfolio: visual rhythm, imagery, typography, and narrative pacing.
- Games/creative tools: expressive, interactive, and polished, without sacrificing controls.

## Visual Direction Checks

- Use one clear palette direction with functional contrast.
- Avoid generic purple/blue gradients, floating blobs, nested cards, and ornamental noise.
- Avoid viewport-scaled text. Make labels fit their containers.
- Use stable dimensions for boards, grids, icon buttons, counters, toolbars, and tiles.
- Prefer icons for common actions; add tooltips where meaning is not obvious.

## Review Checklist

- Does the first screen show the actual product/task, not a marketing placeholder?
- Can a user complete the primary workflow without reading explanatory feature text?
- Are controls discoverable, correctly grouped, and reachable by keyboard?
- Does every dynamic state preserve layout stability?
- Does mobile preserve hierarchy without clipping or overlap?
