# Goal
Apply contemporary layout patterns with intentional hierarchy, rhythm, and interaction rather than generic templates.

# Instructions
1. Select a layout pattern that matches the task: split pane, editorial grid, command center, timeline, kanban, media wall, wizard, comparison, or detail-master.
2. Adapt the pattern to the product context and content density.
3. Define grid behavior, navigation structure, sticky regions, and responsive transformations.
4. Use hierarchy through spacing, type scale, alignment, and content grouping.
5. Include expected states and interactions for the chosen pattern.
6. Verify the layout with real content and edge cases.

# Input
Use the product context, content model, user tasks, aesthetic direction, and framework.

# Output
Generate layout structure, components, CSS, responsive behavior, and interaction states.

# Best Practices
- Do not choose a pattern because it is fashionable; choose it because it serves the workflow.
- Avoid nested card-heavy layouts.
- Keep primary actions visually and spatially connected to their context.
- Use stable dimensions for fixed-format controls and grids.
- Ensure the layout survives long text and localization.
